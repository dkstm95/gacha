# gacha journal

Use when the user wants to record an investment decision, thesis, action conditions, or postmortem.

Workflow:

1. Capture the original user request and decision context.
2. Record current price, data timestamp, source links, and thesis.
3. Record action conditions: buy, watch, stop-loss, trim, take-profit, exit, and invalidation.
4. Record risks and strongest opposite-view objections.
5. For follow-up reviews, compare the original thesis against actual outcomes.

Minimum output:

- decision record
- thesis
- action conditions
- data freshness
- risks and counterarguments
- review date
- source log
