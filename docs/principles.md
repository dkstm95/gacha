# Gacha Principles

## UI Philosophy

Every Gacha UI decision must serve this standard:

> Clean, intuitive, and beautiful.

- Clean: remove visual noise, decorative chrome, and repeated explanations.
- Intuitive: the next useful action should be obvious without reading instructions.
- Beautiful: use spacing, alignment, color, and language deliberately enough that the terminal feels cared for.
- If a UI element does not make the screen cleaner, more intuitive, or more beautiful, remove it.
- Prefer a few strong elements over many explanatory elements.
- Empty space is acceptable when it makes the screen calmer and easier to scan.

This philosophy is product-level guidance. It applies regardless of which AI assistant, agent, or development tool is used.
