# Gacha Principles

## Product Belief

Investment outcomes cannot be predicted perfectly. In the end, every investment
decision carries uncertainty.

Gacha exists because careful research can still improve the odds. The product
should help users turn unclear investment questions into structured decisions by
checking current data, comparing alternatives, surfacing risks, and producing a
clean report.

Gacha must not feel like a random stock picker or a tool that promises certainty.
It should feel like a disciplined research desk for probabilistic decisions.

## Decision Ladder

Gacha should support users at different levels of investment clarity:

1. Discover: the user does not know what or how to invest in yet.
2. Theme selection: the user knows a field or theme, but not the specific stock,
   ETF, asset, or instrument.
3. Entry planning: the user knows what they may want to buy, but not when or
   under what conditions.
4. Holding management: the user owns something and needs hold, trim, sell, or
   stop-out rules.
5. Portfolio review: the user needs allocation, concentration, overlap, or risk
   review across positions.

The UI should let users start from any point on this ladder. The next useful
action should be obvious whether the user arrives with a vague goal, a theme, a
ticker, an existing position, or a full portfolio.

## UI Philosophy

Every Gacha UI decision must serve this standard:

> Clean, intuitive, and beautiful.

- Clean: remove visual noise, decorative chrome, and repeated explanations.
- Intuitive: the next useful action should be obvious without reading instructions.
- Beautiful: use spacing, alignment, color, and language deliberately enough that the terminal feels cared for.
- If a UI element does not make the screen cleaner, more intuitive, or more beautiful, remove it.
- Prefer a few strong elements over many explanatory elements.
- Empty space is acceptable when it makes the screen calmer and easier to scan.

This philosophy is product-level guidance. It applies regardless of which AI assistant, agent, or development tool is used.
